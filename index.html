Welo Data Assistant Audit (Version 3.1.12)
Breakdown of what the extension monitors, verified by audit of its source code.

Welo Data Assistant is a universal tool used by multiple different Welocalize roles. For SQRs, invasive tracking features are off by default, as shown in the code below.

TL;DR Conclusion

Majority of extension functions are currently turned off on RaterHub. It is also important to note that devs could enable full tracking in a future update. I strongly advise against using extensions for RaterHub that are no longer actively supported by their developers.

1. Screen Recording (Sentry Replay)

The extension utilizes Sentry Replay, a standard developer tool used to record screen sessions for bug diagnostics. The source code specifies a 10% probability that any given rating session will be recorded.

Because their text masking configuration is turned OFF (maskAllText: false), the recording captures all visible text and HTML elements on the page. Therefore, if you use an extension that injects custom buttons or timers directly onto RaterHub, those elements will be recorded.

Verdict: Telemetry Enabled (10% normal chance / 100% if JS bug appears)

2. Open Tabs Monitoring

When you finish a task, Welo scans your browser and sends the URLs of all your open tabs directly to their server. It is unclear exactly why they are collecting this data, but the code explicitly triggers it upon task completion.

View Extension Source Code 
async function sendTaskFinishedSignal() {
    let openTabs = [];
    
    // Step 1: Ask Chrome for a list of EVERY open tab
    const tabs = await chrome.tabs.query({});
    
    // Step 2: Extract just the URLs from that list
    openTabs = tabs.map(tab => tab.url);
    
    // Step 3: Send the URLs to the Welo server
    api.post('/user-activity', {
        action: "fulfill",
        tabs: openTabs // <-- This sends your personal tabs!
    });
}

Recommendation: Do not leave personal tabs or any extension settings (chrome://extensions/) page open while working.

Verdict: Actively Tracked.

3. Other Extension Monitoring

While the extension's manifest asks for the management permission, a full search of the source code proves it never actually calls the chrome.management API. It has absolutely no ability to scan your browser, meaning it cannot see that you have RH Toolkit or any other extension installed.

View manifest.json 
{
    "permissions": [
        "tabs",
        "storage",
        "management", // <-- It asks for permission to see other extensions
        "webRequest"
    ]
}

But does it ever use it?
Searching the entire Welo source code for the command chrome.management.getAll() or management.getAll yields 0 Results.

Verdict: NOT tracked.

4. Clipboard & Keyboard Tracking (Copy/Paste)

Inside the extension's source code, the developers hardcoded a "feature flag" object that controls where the copy/paste and keyboard tracking is allowed to run. Because RaterHub is set to false, the extension immediately kills the clipboard and keyboard monitors before they ever start.

View Extension Source Code 
// The hardcoded configuration in the extension
const copyBlockingConfig = {
    ewoq: true,       // <-- Tracking is ENABLED for AQRs
    raterhub: false,  // <-- Completely DISABLED for SQRs
    heartex: false,
    feather: false
};

function startClipboardMonitor() {
    const platform = getCurrentPlatform(); // e.g. 'raterhub'
    
    // The Killswitch:
    // If the platform is 'raterhub', it checks the config and sees 'false'.
    // The word 'return' tells the computer to STOP the function instantly.
    if (!copyBlockingConfig[platform]) {
        return; 
    }
}
Verdict: Completely Disabled. 100% Safe.

5. Mouse Tracking

The extension ignores your general mouse movements, clicks, and scrolling speed. The only mouse-tracking code inside the extension is activated exclusively when you use the "Report a Bug" feature to draw a screenshot rectangle.

View Extension Source Code 
// The Mouse Tracker only exists inside the Bug Reporter User Interface (class: "editor__tool")
const FeedbackScreenshot = {
    setup() {
        // Step 1: You click your mouse down to start drawing the screenshot box
        document.addEventListener("mousedown", function(event) {
            
            // Step 2: IT TURNS ON THE MOUSE TRACKER
            // It only starts watching the mouse *after* you click down to draw
            document.addEventListener("mousemove", drawRectangle);
        });

        // Step 3: You release the mouse button (finished drawing the box)
        document.addEventListener("mouseup", function() {
            
            // Step 4: IT DESTROYS THE MOUSE TRACKER
            // It instantly stops watching your mouse
            document.removeEventListener("mousemove", drawRectangle);
        });
    }
}

If you do not open the "Report a Bug" tool, the editor__tool UI is never created, and it is impossible to trigger this tracking code. Your mouse movements are completely private.

Verdict: NOT tracked.

6. Auto Submit & Button Clicks

Welo does not track physical mouse clicks on the "Submit", "Skip", or "Cancel" buttons. Instead, it monitors the resulting network data you send to RaterHub. Because of this, using an automation tool to programmatically click "Submit" is completely indistinguishable from a real human click.

View Extension Source Code 
// It listens to invisible network requests, NOT physical mouse clicks
chrome.webRequest.onBeforeRequest.addListener(
    function(request) {
        // Step 1: Did the user just submit task data to the RaterHub server?
        if (request.url.includes("/evaluation/rater/task/commit")) {
            // Step 2: If yes, trigger the task finished signal
            sendTaskFinishedSignal();
        }
    },
    { urls: ["https://www.raterhub.com/evaluation/rater/task/*"] }
);
Verdict: 100% Safe to Automate.
