Sentry Privacy Audit
Verification Guide

This guide provides step-by-step instructions on how to locate, access, and verify the Sentry Session Replay configuration settings inside the Welo Data Assistant extension source code.

1. Locating the Extension Source Code

Locate the unpacked extension directory on your system. If you installed it normally via the Chrome Web Store, it is located in your Chrome User Data path under:

%localappdata%\Google\Chrome\User Data\Default\Extensions\oceohhnabfcnjfcpppaegdfecladagcc\3.1.12_0\

Copy the folder to your desktop or open it in a text/code editor (like VS Code or Notepad++). Open the following file in your editor: content-script/index.js

2. Finding the Sentry Configuration

Because the extension file is minified (compressed onto very few lines), it is best to use your editor's "Find/Search" function (Ctrl+F).

Search for the key string replaysSessionSampleRate. This will take you directly to Sentry's initialization block, which configures how your rating sessions are recorded.

Because everything is on one long line, you can also search directly for replaysOnErrorSampleRate:1 or maskAllText:!1 to instantly verify those exact settings exist in the code. (Note: The minifier turns false into !1 and true into !0 to save space).

3. The Code Proof

Here is the exact Sentry configuration block found in content-script/index.js, formatted for readability:

View content-script/index.js 
// Sentry Initialization inside Welo's content-script/index.js
p.init({
  dsn: f, // Points to Welo's self-hosted server: sentry.welocalize.tools
  environment: Ui.environment, // "production"
  release: ...,
  tracesSampleRate: 1,
  replaysSessionSampleRate: .1, // 10% of sessions are recorded
  replaysOnErrorSampleRate: 1, // 100% of sessions with a JS error are recorded
  sendDefaultPii: !1,
  skipBrowserExtensionCheck: !0,
  integrations: [
    p.browserTracingIntegration(),
    p.replayIntegration({
      maskAllText: !1, // Page text and task content are NOT masked (!1 means false)
      maskAllInputs: !0, // Text typed into input fields IS masked (!0 means true)
      blockAllMedia: !1
    })
  ],
  ...
})

4. Configuration Impact

Here is what these configurations mean for the extension:

  • replaysSessionSampleRate: .1
    The extension rolls a 10% random chance (1 in 10) on every new session to record your screen interactions.
  • replaysOnErrorSampleRate: 1
    If any JavaScript error occurs in your browser while rating (even minor, unrelated page errors), the extension immediately triggers a 100% capture of the session and uploads the recording.
  • maskAllText: !1 (false)
    Because this is set to false, Sentry captures all visible text on the rating page (e.g., task queries, project guidelines, and injected DOM elements from other extensions) in plain text.
  • maskAllInputs: !0 (true)
    Any text you manually type into input fields, textareas, or text-boxes is masked (hidden) and will not be visible in the recording.
  • Self-Hosted Telemetry Server
    The destination URL is sentry.welocalize.tools. This confirms that all recorded telemetry is sent to private servers, rather than Sentry's public cloud.

5. The DOM Injection Risk

Welo's extension does NOT scan your browser for other installed extensions. However, it can still "see" them buttons or anything injected on the screen through Sentry Replay.

  • Mechanism: Sentry does not record a video file. Instead, it uses the rrweb library to snapshot the webpage's HTML structure (DOM) and tracks every change using a MutationObserver.
  • Injection Flaw: If an extension injects custom buttons or timers into the RaterHub webpage, rrweb captures that new HTML code.
  • Risk: The developer's dashboard reconstructs this HTML into a visual playback. Because maskAllText is false, if you are unlucky enough and someone checks your session, they will clearly see your tools rendered on screen.

6. Continuous Upload Behavior

It is important to understand that Sentry does NOT wait until you finish a task to upload the recording.

  • Near Real-Time: Sentry packages your screen activity into small chunks and uploads them continuously in the background (usually every 5 seconds) while you work.
  • Closing the Tab: If you close the RaterHub tab or navigate away, the extension immediately triggers a final upload (p.flush(500)) to ensure the very last seconds of your session are successfully sent to the server.

TL;DR Conclusion

Majority of extension functions are currently turned off on RaterHub. It is also important to note that devs could enable full tracking in a future update. I strongly advise against using extensions for RaterHub that are no longer actively supported by their developers. Thank you for reading!

1. Screen Recording (Sentry Replay)

The extension utilizes Sentry Replay, a standard developer tool used to record screen sessions for bug diagnostics. The source code specifies a 10% probability that any given rating session will be recorded.

Because their text masking configuration is turned OFF (maskAllText: false), the recording captures all visible text and HTML elements on the page. Therefore, if you use an extension that injects custom buttons or timers directly onto RaterHub, those elements will be recorded.

2. Open Tabs Monitoring

When you finish a task, Welo scans your browser and sends the URLs of all your open tabs directly to their server. It is unclear exactly why they are collecting this data, but the code explicitly triggers it upon task completion.

View Extension Source Code 
async function sendTaskFinishedSignal() {
    let openTabs = [];
    
    // Step 1: Ask Chrome for a list of EVERY open tab
    const tabs = await chrome.tabs.query({});
    
    // Step 2: Extract just the URLs from that list
    openTabs = tabs.map(tab => tab.url);
    
    // Step 3: Send the URLs to the Welo server
    api.post('/user-activity', {
        action: "fulfill",
        tabs: openTabs // <-- This sends your personal tabs!
    });
}

Recommendation: Do not leave personal tabs or any extension settings (chrome://extensions/) page open while working.

Verdict: Actively Tracked.

3. Other Extension Monitoring

While the extension's manifest asks for the management permission, a full search of the source code proves it never actually calls the chrome.management API. It has absolutely no ability to scan your browser, meaning it cannot see that you have RH Toolkit or any other extension installed.

Searching the entire Welo source code for the command chrome.management.getAll() or management.getAll yields 0 Results.

Verdict: NOT tracked.

4. Clipboard & Keyboard Tracking (Copy/Paste)

Inside the extension's source code, the developers hardcoded a "feature flag" object that controls where the copy/paste and keyboard tracking is allowed to run. Because RaterHub is set to false, the extension immediately kills the clipboard and keyboard monitors before they ever start.

View Extension Source Code 
// The hardcoded configuration in the extension
const copyBlockingConfig = {
    ewoq: true,       // <-- Tracking is ENABLED for AQRs
    raterhub: false,  // <-- Completely DISABLED for SQRs
    heartex: false,
    feather: false
};

function startClipboardMonitor() {
    const platform = getCurrentPlatform(); // e.g. 'raterhub'
    
    // The Killswitch:
    // If the platform is 'raterhub', it checks the config and sees 'false'.
    // The word 'return' tells the computer to STOP the function instantly.
    if (!copyBlockingConfig[platform]) {
        return; 
    }
}
Verdict: Completely Disabled. 100% Safe.

5. Mouse Tracking

The extension ignores your general mouse movements, clicks, and scrolling speed. The only mouse-tracking code inside the extension is activated exclusively when you use the "Report a Bug" feature to draw a screenshot rectangle.

View Extension Source Code 
// The Mouse Tracker only exists inside the Bug Reporter User Interface (class: "editor__tool")
const FeedbackScreenshot = {
    setup() {
        // Step 1: You click your mouse down to start drawing the screenshot box
        document.addEventListener("mousedown", function(event) {
            
            // Step 2: IT TURNS ON THE MOUSE TRACKER
            // It only starts watching the mouse *after* you click down to draw
            document.addEventListener("mousemove", drawRectangle);
        });

        // Step 3: You release the mouse button (finished drawing the box)
        document.addEventListener("mouseup", function() {
            
            // Step 4: IT DESTROYS THE MOUSE TRACKER
            // It instantly stops watching your mouse
            document.removeEventListener("mousemove", drawRectangle);
        });
    }
}

If you do not open the "Report a Bug" tool, the editor__tool UI is never created, and it is impossible to trigger this tracking code. Your mouse movements are completely private.

Verdict: NOT tracked.

6. Auto Submit & Button Clicks

Welo does not track physical mouse clicks on the "Submit", "Skip", or "Cancel" buttons. Instead, it monitors the resulting network data you send to RaterHub. Because of this, using an automation tool to programmatically click "Submit" is completely indistinguishable from a real human click.

View Extension Source Code 
// It listens to invisible network requests, NOT physical mouse clicks
chrome.webRequest.onBeforeRequest.addListener(
    function(request) {
        // Step 1: Did the user just submit task data to the RaterHub server?
        if (request.url.includes("/evaluation/rater/task/commit")) {
            // Step 2: If yes, trigger the task finished signal
            sendTaskFinishedSignal();
        }
    },
    { urls: ["https://www.raterhub.com/evaluation/rater/task/*"] }
);
Verdict: 100% Safe to Automate.
