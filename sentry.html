Sentry Privacy Audit
Verification Guide

This guide provides step-by-step instructions on how to locate, access, and verify the Sentry Session Replay configuration settings inside the Welo Data Assistant extension source code.

1. Locating the Extension Source Code

Locate the unpacked extension directory on your system. If you installed it normally via the Chrome Web Store, it is located in your Chrome User Data path under:

%localappdata%\Google\Chrome\User Data\Default\Extensions\oceohhnabfcnjfcpppaegdfecladagcc\3.1.12_0\

Copy the folder to your desktop or open it in a text/code editor (like VS Code or Notepad++). Open the following file in your editor: content-script/index.js

2. Finding the Sentry Configuration

Because the extension file is minified (compressed onto very few lines), it is best to use your editor's "Find/Search" function (Ctrl+F).

Search for the key string replaysSessionSampleRate. This will take you directly to Sentry's initialization block, which configures how your rating sessions are recorded.

Because everything is on one long line, you can also search directly for replaysOnErrorSampleRate:1 or maskAllText:!1 to instantly verify those exact settings exist in the code. (Note: The minifier turns false into !1 and true into !0 to save space).

3. The Code Proof

Here is the exact Sentry configuration block found in content-script/index.js, formatted for readability:

View content-script/index.js 
// Sentry Initialization inside Welo's content-script/index.js
p.init({
  dsn: f, // Points to Welo's self-hosted server: sentry.welocalize.tools
  environment: Ui.environment, // "production"
  release: ...,
  tracesSampleRate: 1,
  replaysSessionSampleRate: .1, // 10% of sessions are recorded
  replaysOnErrorSampleRate: 1, // 100% of sessions with a JS error are recorded
  sendDefaultPii: !1,
  skipBrowserExtensionCheck: !0,
  integrations: [
    p.browserTracingIntegration(),
    p.replayIntegration({
      maskAllText: !1, // Page text and task content are NOT masked (!1 means false)
      maskAllInputs: !0, // Text typed into input fields IS masked (!0 means true)
      blockAllMedia: !1
    })
  ],
  ...
})

4. Configuration Impact

Here is what these configurations mean for the extension:

  • replaysSessionSampleRate: .1
    The extension rolls a 10% random chance (1 in 10) on every new session to record your screen interactions.
  • replaysOnErrorSampleRate: 1
    If any JavaScript error occurs in your browser while rating (even minor, unrelated page errors), the extension immediately triggers a 100% capture of the session and uploads the recording.
  • maskAllText: !1 (false)
    Because this is set to false, Sentry captures all visible text on the rating page (e.g., task queries, project guidelines, and injected DOM elements from other extensions) in plain text.
  • maskAllInputs: !0 (true)
    Any text you manually type into input fields, textareas, or text-boxes is masked (hidden) and will not be visible in the recording.
  • Self-Hosted Telemetry Server
    The destination URL is sentry.welocalize.tools. This confirms that all recorded telemetry is sent to private servers, rather than Sentry's public cloud.

5. The DOM Injection Risk

Welo's extension does NOT scan your browser for other installed extensions. However, it can still "see" them visually through Sentry Replay.

  • Mechanism: Sentry does not record a video file. Instead, it uses the rrweb library to snapshot the webpage's HTML structure (DOM) and tracks every change using a MutationObserver.
  • Injection Flaw: If an extension injects custom buttons or timers into the RaterHub webpage, rrweb captures that new HTML code.
  • Risk: The developer's dashboard reconstructs this HTML into a visual playback. Because maskAllText is false, if you are unlucky enough and someone checks your session, they will clearly see your tools rendered on screen.

6. Continuous Upload Behavior

It is important to understand that Sentry does NOT wait until you finish a task to upload the recording.

  • Near Real-Time: Sentry packages your screen activity into small chunks and uploads them continuously in the background (usually every 5 seconds) while you work.
  • Closing the Tab: If you close the RaterHub tab or navigate away, the extension immediately triggers a final upload (p.flush(500)) to ensure the very last seconds of your session are successfully sent to the server.
